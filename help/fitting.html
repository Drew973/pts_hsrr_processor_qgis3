Fitting

section_changes table

The section_changes table contains the following columns

  • sec:section label. Needs to be in network. Empty string ('') where gap or not in network.
  • reversed: if survey is in opposite direction to section.
  • xsp
  • run:needs to be in run_info.
  • ch:chainage where run enters section.
  • note:note for user. Has no effect.
  • start_sec_ch,end_sec_ch: Section chainages(in direction of section) where run enters/leaves section. Can be used to calculate section chainage.
  • pk(not displayed):primary key.
  • run(not displayed).
  • pt:point geometry(not displayed).

Readings from ch to next ch are fit to sec,reversed,xsp. Altering the section_changes table (through any means) will update the fitted,resized and coverage tables.

The fitting tab and menu contain tools for viewing and editing it.

Manually add rows through fitting,add row...

Drop rows by right clicking, delete selected rows.

Autofit

Autofit automatically creates the route for the selected run. It is likely to make mistakes where:

  • The nearest section to a reading is not the correct one
  • The gps is inaccurate
  • The network is inaccurate

Autofit does not work out xsp.

Section Chainage

Section chainage is usually calculated from the nearest point on the section to the reading.

Where start_sec_ch and end_sec_ch are set and >0 section chainage is calculated by interpolating between these.

This should be done where the nearest point of the section is not at the correct chainage.

This is important where the GPS has lost signal and for small roundabouts (that readings are closer to wrong side of).
